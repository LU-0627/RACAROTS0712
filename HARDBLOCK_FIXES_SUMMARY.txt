# 硬阻塞问题修复总结

## 修改文件列表
1. models/build.py
2. run_rd_carots.py
3. scripts/server/00_check_environment.sh
4. scripts/server/06_run_synthetic_gpu_smoke.sh
5. scripts/server/RUN_ALL_SERVER_GPU.sh

## 硬阻塞1: Device链路修复
**问题**: torch.cuda.is_available()自动选择，允许自动降级CPU
**修复**:
- models/build.py: 移除自动检测，device从cfg.DEVICE或cfg.RDCAROTS.DEVICE读取
- models/build.py: CUDA不可用时raise RuntimeError，不降级
- models/build.py: build_model(cfg, device)传递device给RDCAROTS(cfg, device=device)
- run_rd_carots.py: 新增--device参数，默认cuda:0
- run_rd_carots.py: load_config返回(cfg, device)
- run_rd_carots.py: 所有mode函数接收device参数

## 硬阻塞2: RDCAROTS测试入口修复
**问题**: 测试调用CAROTS Predictor，无4分量输出，无frozen/guarded_online逻辑
**修复**:
- run_rd_carots.py: 新增_test_rdcarots()函数
- run_rd_carots.py: 检测model.startswith('RDCAROTS')时调用RDCAROTS专用测试
- run_rd_carots.py: frozen模式使用torch.no_grad()不更新
- run_rd_carots.py: guarded_online检查score和confidence条件，不使用label
- run_rd_carots.py: 输出score_embed/pred/delay/uncertainty/final/regime/label
- run_rd_carots.py: 调用RDCAROTSScorer.compute_scores()
- run_rd_carots.py: 保存window_scores.csv和update_log.csv

## 硬阻塞3: 服务器脚本格式修复
**问题**: 脚本单行、无错误处理、可能联网
**修复**:
- 所有脚本第一行: #!/usr/bin/env bash
- 所有脚本第二行: set -euo pipefail
- 00_check_environment.sh: 检查CUDA但不安装
- 06_run_synthetic_gpu_smoke.sh: 默认DEVICE=cuda:0
- 06_run_synthetic_gpu_smoke.sh: CUDA不可用直接exit 1
- 06_run_synthetic_gpu_smoke.sh: 传递--device到run_rd_carots.py
- RUN_ALL_SERVER_GPU.sh: 顺序执行检查和smoke test
- 所有脚本: 日志写入${RESULT_ROOT}/logs
- 所有脚本: 无联网命令、无pip install
